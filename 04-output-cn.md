# 维度四：智能体输出（Output）

> 回答：这个 Skill 执行完毕后，产生了什么？交付物是什么？放在哪里？

---

## 输出分类

### 1. 直接交付物（Direct Deliverable）
用户期望得到的结果，例如：
- 文档（8D 报告、质量分析报告）
- 演示文稿（PPT）
- 表格（Excel、CSV）
- 代码文件（Python、JavaScript）
- 视频、图片

### 2. 过程产物（Process Artifacts）
Skill 执行过程中生成的中间文件：
- `log.md` — 运行日志（每个实例文件夹内）
- `Report.md` — 执行总结/复盘报告
- `Error.md` — 错误记录（若发生错误）
- `TMP/` — 临时文件（执行完毕应清理）

### 3. 可更新的作业指导（Updatable Artifacts）
Skill 执行后可沉淀更新的文件：
- `SKILL.md` — Skill 本身（发现新场景/新边界时更新）
- 对话模板（根据用户反馈优化）
- 运行脚本（发现更优实现时更新）
- 数据结构定义（Schema 演进）
- 知识库条目（知识升级/新增）

---

## 文件命名规范

| 文件类型 | 命名规则 | 示例 |
|----------|----------|------|
| 实例运行文件夹 | `YYYY-MM-DD-task-<序号>` | `2026-05-09-task-001` |
| 运行日志 | `log.md` | 同上目录内 |
| 执行总结 | `Report.md` | 同上目录内 |
| 错误记录 | `Error.md` | 同上目录内 |
| 临时文件 | 放在 `TMP/` 子目录 | `TMP/ocr_p1.png` |
| 最终交付物 | `<项目名>-<产物名>.<ext>` | `8D-BBP343-Report.docx` |

---

## 输出交付流程

```
Skill 执行完成
    │
    ▼
生成过程产物（log.md, Report.md, Error.md）
    │
    ▼
交付物写入用户指定路径 / 默认工作目录
    │
    ▼
向用户呈现结果摘要 + 文件路径
    │
    ▼
[若涉及 Approval Gate] → 等待用户确认 → 执行发布/发送等后续操作
```

---

## 输出质量要求

| 验收项 | 要求 |
|--------|------|
| 完整性 | 所有声明的交付物均已生成 |
| 可读性 | 输出格式正确，字符编码为 UTF-8 |
| 可追溯 | 过程产物（log.md）记录了完整的执行路径 |
| 幂等性 | 相同输入再次执行，应生成相同输出 |
